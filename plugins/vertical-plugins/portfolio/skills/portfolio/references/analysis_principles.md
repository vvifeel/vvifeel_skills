# 분석 원칙 상세

## 1. 질문 의도 해석 원칙

### 의도 해석 → 답변 구조 결정

| 조건 | 답변 구조 |
|---|---|
| 특정 투자조건 필터 + 외부 정보 | 기업별 `###` 섹션 + 뉴스 상세(건당 3~5줄) + 투자 관점 시사점 |
| 외부 정보 대상 5개사 이하 | 기업별 개별 서술 (표 생략 가능) |
| 외부 정보 대상 6개사 이상 | 표로 목록 먼저 → 기업별 서술 |
| 집계 수치만 필요 | 표 + 차트 중심, 서술 최소화 |
| "간단히", "요약만", "리스트만" 명시 | 위 규칙 무시하고 요청 형태 그대로 |

### 외부 정보 요청의 목적 판단

**심층 모니터링** (핵심 포트폴리오 기업군 + 뉴스):
- BoD Observer, Pro-rata 등 특정 투자조건으로 필터된 기업군
- 당사가 깊이 관여하는 기업들 → 놓친 변화 점검 목적
- → 업체별 섹션 구분 + 뉴스 상세 서술 + 투자 관점 코멘트

**빠른 스캔** (일반 현황 파악):
- 전체 포트폴리오 또는 대규모 그룹 대상
- → 헤드라인 + 1~2줄 요약

### 적용 예시

**예시 1 — BoD Observer + 뉴스**
- 표면: "BoD Observer 업체 + 최근 1개월 뉴스"
- 의도: 이사회 수준 관여 기업 → 심층 모니터링
- 구조: 업체별 `###` → 뉴스 건별 상세(날짜 + 3~5줄) + 시사점

**예시 2 — 섹터별 집계**
- 표면: "섹터별 투자 건수"
- 의도: 포트폴리오 구성 현황 파악
- 구조: 집계 표 + 가로 막대 차트 + 간략 코멘트

**예시 3 — 특정 기업 히스토리**
- 표면: "A사 투자 히스토리"
- 의도: 투자 경위 확인 또는 후속 검토 준비
- 구조: 라운드별 표 + 현황(상태·CEO) + 멀티라운드면 투자 흐름 한 줄 코멘트

## 2. 외부 정보 조회 우선순위

> 웹검색을 CB Insights보다 먼저 실행하는 것은 **금지**.

### 조회 순서

```
1순위: CB Insights
2순위: 웹검색 (CB Insights에 해당 정보 없을 때만)
```

### CB Insights 도구별 용도

| 정보 유형 | 도구 | 비고 |
|---|---|---|
| 최근 뉴스·동향 | `get_company_news` | 기사 날짜 필수 확인 |
| 현재 기업가치 | `get_company_profile` / `get_company_funding` | 투자 당시 대비 배수 산출 가능 |
| 최근 펀딩 라운드 | `get_company_funding` | |
| 경쟁사 현황 | `get_company_competitors` | |
| 기업 개요 | `get_company_profile` | |
| 투자자 구성 | `get_company_investors` | |

### Fallback 규칙

- CB Insights 결과 없음 → 웹검색으로 재조회 → `(출처: 웹검색)` 명시
- 일부만 확인 → 확인된 부분은 `(출처: CB Insights)`, 나머지는 `(출처: 웹검색 보완)`

### CB Insights 연동 제안 기준

**제안하는 경우:**
- 특정 기업 현황·경쟁력·시장 포지션 파악이 핵심
- 외부 펀딩 동향, 경쟁사 비교, 투자자 구성 등 DB 외 데이터 필요
- 현재 기업가치 확인 필요 (배수 산출 목적 포함)

**제안하지 않는 경우:**
- `get_portfolio_summary` 단독 호출 결과
- 집계·통계만 반환된 경우
- 단순 목록 확인 목적
- 직전 응답에서 사용자가 거절한 경우

**5개사 초과 시:** 전체 조회 여부 먼저 확인 후 진행
