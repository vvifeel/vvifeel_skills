# rules-symbol.md — 풋노트·반복구조·기호·날짜·표현 통일 규칙

---

## STEP 10. 풋노트 규칙 (STRICT)

```
구현: 인라인 파랑 단락 (inject에서 자동 처리)
형식: "* 내용" (inject 자동 추가), 마침표 없음
```

### 풋노트 길이 규칙

```
2줄 절대 금지 — 풋노트는 반드시 1줄 이내 완결 (winword STRICT)

초과 위험 시 처리 순서:
  1. 자간 조절: tcs(s, -8) → tcs(s, -12) (최대 0.6pt)
  2. 설명 텍스트 단축: 핵심 의미만 남기고 압축
  → 어떤 수단을 써도 1줄을 넘길 수밖에 없으면 설명을 더 짧게 재작성 (길이 우선)

목표 길이 기준 (참고):
  윗줄 환산 25자 이상 → 풋노트 약 16자 이내 안전
  윗줄 환산 20자 이하 → 풋노트 약 12자 이내 안전
  → 정확한 기준은 Word COM 렌더링으로 확인
```

### 배치 원칙

```
기본값: 위치가 아니라 필요성 기준. 필요한 줄이면 삽입하고, 필요 없으면 삽입하지 않는다.
허용: lv2·lv3·lv4 문장에 대한 풋노트 (본문 어디서든 필요하면 삽입)
금지: lv1(H1) 단락 전체에 대한 풋노트
금지: 단락 전체를 설명하는 풋노트 (특정 문장 아닌 섹션 요약형)
금지: 위치 균형을 맞추기 위한 억지 풋노트 추가

배치 위치:
  - 풋노트가 필요한 줄 바로 아래 단락으로 삽입
  - 설명 대상과 풋노트 사이에 빈줄 삽입 금지
  - * 위치: 반드시 _isFn anchor로 지정 — 풋노트가 필요한 단어 시작 바로 아래에 * 배치 (STRICT)
    "기호 시작점+2칸" 기본값 사용 금지 — anchor 없는 fn() 단독 사용 금지
    예) { lv: LV.GAP, _isFn: { anchor: "5.1%", text: "임금 인상률 산식 기준" } }
    → anchor는 해당 단어의 열 위치를 계산하는 데 사용되며 최종 출력은 "* 임금 인상률 산식 기준"만 표시
  - 특정 항목이 여러 문장에 반복 등장하는 경우:
    → 해당 항목이 처음 등장하는 위치 아래에만 풋노트 배치
    → 이후 반복 등장 시 추가 풋노트 금지
  - 같은 설명 대상이 여러 위치에 있으면 더 먼저 등장하거나 독자가 처음 해석해야 하는 위치에만 배치

풋노트가 적합한 상황 (해당되면 넣되 강제 아님):
  - 약어·전문용어 첫 등장 시 (예: IRR, CAGR, NPS 등)
  - 수치의 출처·기준·시점이 명시 필요한 경우
  - 주석이 없으면 독자가 오해할 수 있는 내용
  - 부연 설명이 필요하지만 인라인으로 추가하면 한 줄을 초과하는 경우

defs 배열 패턴 — 본문 h3 풋노트:
  { lv: LV.BODY, text: "- 설명 문장 (IRR 등 첫 등장)" },
  { lv: LV.GAP,  _isFn: "IRR: 내부수익률, 투자 수익률 지표" },  // 윗줄 28자 기준 19자 이내
  { lv: LV.BODY, text: "- 다음 문장" },
  // IRR이 다시 등장해도 _isFn 추가 금지

defs 배열 패턴 — 특정 위치 anchor 풋노트:
  { lv: LV.BODY, text: "- OOOO 이 있는 OOO 가 존재" },
  { lv: LV.GAP,  _isFn: { anchor: "OOO", text: "설명" } },
  // 출력: "* 설명"만 표시되며, OOO는 풋노트 본문에 자동 반복하지 않음

금지 패턴:
  - 본문 풋노트 개수를 맞추려고 필요 없는 용어 설명을 추가
  - 같은 설명 대상을 여러 위치에 중복 배치

SUB 두 줄(SUB+SUB2) 항목에는 풋노트 붙이지 않음

별첨 언급:
  - 별첨은 본문 이해에 직접 필요한 경우에만 언급한다.
  - 본문 흐름상 참조가 불필요하면 별첨을 본문에서 언급하지 않는다.
  - 언급이 필요한 경우에도 본문에는 "별첨"이라는 단어를 직접 쓰지 않는다.
  - 본문에는 별첨 제목만 자연스럽게 언급한다.
  - 별첨 번호는 해당 제목 위치 아래 풋노트로만 표시한다.
  예)
    { lv: LV.BODY, text: "- 점검 항목 요약표를 활용하면 자체 점검에 재사용 가능" },
    { lv: LV.GAP,  _isFn: { anchor: "점검 항목 요약표", text: "별첨1" } },
```

## STEP 10-B. 반복 구조 계층 패턴

보고서에서 기업별·종목별·사업부별 등 **동일 항목이 반복**되는 경우 계층 구조:

```
반복 단위 → lv2(□)
반복 단위의 세부 항목 → lv3(-) 또는 원형번호·※
세부 항목의 내용 → lv4(·) 또는 원형번호·※

예) 기업별 상세 현황:
  □ A기업        ← lv2 (반복 단위)
    - 기업현황   ← lv3
      · 내용     ← lv4
    - 최신동향   ← lv3
      · 내용     ← lv4
    - 검토의견   ← lv3
      · 내용     ← lv4
  □ B기업        ← lv2
    ...

금지: 반복 단위의 세부 항목(현황·동향·의견 등)을 반복 단위(기업명)와 같은 lv2로 나열
→ lv2가 5개 이상 연속되면 반복 구조인지 먼저 검토

계층 리듬:
  - lv2별 lv3 개수는 2~5개 범위에서 맥락에 따라 달리한다.
  - 모든 lv2를 lv3 3개씩 맞추지 않는다. 균일한 3개 bullet 반복은 작위적으로 보일 수 있다.
  - lv3는 목적에 맞는 핵심 판단문을 먼저 만들고, 한 줄 상한 안에서는 내용을 충분히 담는다.
  - 하한 없음. 글자 수를 특정 구간에 맞추지 말고, 상한 초과 시에만 자간·표현 조정을 적용한다.
  - 같은 맥락의 현황·원인·영향·판단은 짧은 여러 줄로 분해하지 말고 한 줄 안에서 결합한다.
  - 분리는 정보 성격이 달라지거나 상한 초과로 결합이 어려울 때만 사용한다.
  - 본문·제목·표 셀의 여러 항목 나열은 가운데 도트(·) 대신 쉼표(,)를 사용한다.

lv4 선택 활용:
  - 모든 lv3에 lv4를 붙이지 않는다.
  - 근거·예시·리스크·후속조치가 필요한 lv3에만 1~2개 lv4를 붙인다.
  - lv3가 단순 나열로 길어지면 lv4(·)로 분리하여 계층 압축한다.
  - lv2→lv3→lv4 3단계로 충분히 표현 가능한 경우 lv2 남용 금지

원형번호·※ 사용:
  - lv2로 사용 금지. □ 소제목을 대체하지 않는다.
  - lv3 위치가 맞으면 LV.NUM3로 작성하고 body(-)와 동일한 들여쓰기·여백을 따른다.
  - lv4 위치가 맞으면 LV.NUM4로 작성하고 sub(·)와 동일한 들여쓰기·여백을 따른다.
  - 같은 블록 안에서 -/·와 원형번호를 섞지 않는다.
```

## STEP 11. 기호·날짜·표현 통일 (STRICT)

### 계층 기호 — 허용 목록

```
lv1:  1. 2. 3.  ← 반드시 아라비아 숫자 + 마침표 형식 (Ⅰ. Ⅱ. / I. II. / 가. 나. 절대 금지)
lv2:  □
lv3:  -  (하이픈) 또는  ① ② ③ ...  또는  ※
lv4:  ·  (도트)   또는  ① ② ③ ...  또는  ※

lv1 번호 표기 규칙 (STRICT):
  h1() 헬퍼에 전달하는 text는 반드시 "1. 제목", "2. 제목", "3. 제목" 형식
  로마자(Ⅰ Ⅱ Ⅲ, I II III), 가나다, 괄호번호((1)(2)) 절대 금지
  번호는 자동 생성하지 않음 — defs 배열에 직접 작성
  올바른 예: { lv: LV.H1, text: "1. 개요" }
  금지 예:   { lv: LV.H1, text: "Ⅰ. 개요" }  { lv: LV.H1, text: "I. 개요" }

원형번호(①②③)와 ※ 는 lv3·lv4 위치에서만
하이픈(-) · 도트(·) 를 대신하여 사용 가능.
lv2에서는 사용 금지 — □ 소제목만 허용.
본문·제목·표 셀의 나열 구분자로 가운데 도트(·) 사용 금지 — A, B, C처럼 쉼표로 나열.

화살표류: → ← ↑ ↓ ⇒ 등 방향 표시 목적으로 문장 내 사용 가능 (계층 기호 대체 불가)

지양 (정말 필요한 경우만 예외):
  별표(★☆✱), 전화(☎), 손가락(☞☜), 하트(♥), 클럽(♣) 등
  문서에서 잘 사용되지 않는 특수기호

금지:
  ○ ● ▶ ▷ ▪ • ◆ ◇ 등 계층 기호와 혼동되는 기호
  기호 없는 body/sub 레벨 항목 — 업체명·기업명·고유명사도 예외 없음
    예) 기호 없이 "Carbon Health" 단독 등장 → 금지
        "① Carbon Health" 또는 "- Carbon Health" → 허용
  내용 없는 기호 단락
  lv5 이하 계층 — 최대 lv4 까지만 사용
```

### 기호 혼용 금지 규칙 (STRICT)

```
같은 레벨(같은 들여쓰기)에서 기호를 섞으면 안 됨.
한 블록(같은 lv2 아래 같은 레벨) 내에서는 하나의 기호만 사용.

[올바른 사용 — lv3 위치에서 원형번호]
  □ lv2 제목
    ① 항목 A          ← lv3 위치, 하이픈 대신 원형번호
    ② 항목 B
    ③ 항목 C

[올바른 사용 — lv4 위치에서 ※]
  □ lv2 제목
    - lv3 항목
      ※ 보충 설명     ← lv4 위치, 도트 대신 ※

[올바른 사용 — lv3 위치에서 ※]
  □ lv2 제목
    ※ 주의사항 설명   ← lv3 위치, 하이픈 대신 ※

[금지 — 같은 레벨 혼용]
  □ lv2 제목
    - 항목 A           ← 하이픈
    ① 항목 B           ← 원형번호  → 금지 (같은 lv3 위치에 혼용)
    - 항목 C

[금지 — 같은 레벨 혼용]
  □ lv2 제목
    - 항목 A
      · 세부 내용      ← 도트
      ※ 보충 설명     ← ※  → 금지 (같은 lv4 위치에 혼용)
```

### ※ · 원형번호 헬퍼 사용법

```javascript
// lv2 위치 사용 금지: □ 소제목만 허용

// lv3 위치: NUM3 사용 (하이픈 대신)
// defs: { lv: LV.NUM3, text: "① 항목 A" }
num3("① 항목 A")    // 하이픈 대신 원형번호 (lv3)
num3("※ 주의사항")  // 하이픈 대신 ※ (lv3)

// lv4 위치: NUM4 사용 (도트 대신)
// defs: { lv: LV.NUM4, text: "① 세부 항목" }
num4("① 세부 항목")  // 도트 대신 원형번호 (lv4)
num4("※ 보충 설명")  // 도트 대신 ※ (lv4)

// ⚠️ hanging 값: num3()=HANG, num4()=HANG — 해당 레벨의 기본 룰과 동일
// ⚠️ 같은 lv2 블록 내 같은 레벨에서 -/· 와 ①/※ 혼용 금지
```

### lv2 원형번호 금지 예시

```
[금지] lv2 위치에 원형번호 사용
  □ 배경         ← h2()
  ① 항목 A      ← 금지 (lv2가 아니라 lv3 또는 lv4로 내려야 함)

[허용] lv3 위치에서 원형번호 사용
  □ 배경
    ① 항목 A    ← LV.NUM3
    ② 항목 B    ← LV.NUM3
```

### 날짜·연도
| 상황 | 형식 | 예시 |
|------|------|------|
| 문서 작성일 (타이틀 아래) | YY.MM.DD(요일), 연도 앞 ' 금지 | 26.05.15(목) |
| 본문·풋노트 전체 날짜 | 'YY.MM.DD | '26.03.28 |
| 연도+월 | 'YY.M / 'YY.MM | '26.5, '26.12 |
| 연도 단독 | 'YY년 | '26년 ('24년) |
| 분기 | 'YY.NQ | '26.3Q |
| 기간·범위 (물결) | 좌우 공백 1칸 | '24년 ~ '26년, 3.20 ~ 4.20 |

금지: 본문·풋노트에서 "2026년 3월 28일", "2026.03.28" 형식 사용

기호 공백:
  - `~`, `+` 기호는 좌우 공백을 1칸씩 둔다.
  - `3.20.~4.20` 금지 → `3.20 ~ 4.20`
  - `A+B` 금지 → `A + B`
  - 단항 부호(예: +3%)처럼 한쪽에 비교 대상이 없는 경우는 예외

### 본문·풋노트 문체

```
원칙: 보고서 목적에 맞는 판단을 먼저 세우고, 한 줄 상한 안에서는 내용을 충분히 담는다.
짧은 문장을 먼저 만들지 말고, 목적·역할·핵심 판단을 정한 뒤 상한 초과 시에만 조사·중복어·서술어를 줄인다.
단, 의미가 흐려지거나 문장이 부자연스러워지는 경우 필요한 조사는 허용한다.

권장:
  - 직전 문맥에서 주어가 명확하면 다시 쓰지 않음
  - 최우선: 명사형·판단명사 종결
  - 금지: "~이다/~로 보인다/~가 필요하다" 등 문장완결형 종결
  - 차선: "~함/있음/됨/해야 함"류는 문장완결형보다 낫지만 반복 사용 지양
  - 글자 수 하한을 맞추기 위한 인위적 압축 금지
  - 한 줄 상한 안에서는 원인·근거·판단을 함께 담아 짧은 메모형 단락 남발 방지
  - 주어+조사 단문은 명사구, 관형절, 상황절, 판단명사형 등으로 다양하게 재구성
  - 같은 변환 패턴을 반복하지 않음. 예시는 방향만 참고하고 문맥에 맞게 변주
  - 풋노트는 특히 주어 반복 없이 기준·범위·포함 항목 중심으로 작성
  - 한 줄 기준 +1~4자 미세 초과는 내용 변경 전에 자간(cs=-8 또는 -12) 우선 적용

상한 초과 시 조정 순서:
  1. 줄의 역할 결정: 현황, 원인, 영향, 판단, 조치, 리스크
  2. 목적에 맞는 핵심 판단 초안 작성
  3. 한 줄 상한 이내면 그대로 유지
  4. 너무 짧은 단락이 연속되면 같은 맥락의 원인·근거·판단을 한 줄 안에서 결합
  5. 초과 시 조사·중복어·불필요 수식어 제거
  6. 필요한 경우 서술어를 판단명사로 압축
  7. 미세 초과는 자간 적용, 자간 한도 초과 시에만 의미 단위 분리
```
