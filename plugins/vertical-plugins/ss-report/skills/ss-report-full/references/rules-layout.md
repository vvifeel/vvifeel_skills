## STEP 6. 표 페이지 처리 (STRICT)

**표는 항상 단일 페이지 내에 완전히 위치해야 한다.**

```
표 분리 방지 방법:
  1. mkRow에 cantSplit:true 적용 (기본 설정)
  2. 표가 페이지 경계에 걸릴 것으로 예상되면:
     → 표 직전 내용의 분량을 조절하여 표 전체가 다음 페이지로 내려가도록
     → 직전 내용 압축: 1~3줄 줄이기 → 표가 현재 페이지에 들어오도록
     → 직전 내용 보강: 1~2줄 늘리기 → 표가 다음 페이지로 완전히 이동

표 크기 추산 (행당 약 400~500 DXA, 헤더 포함):
  3행 표 ≈ 1,500~2,000 DXA
  5행 표 ≈ 2,500~3,000 DXA
  → JS 코드 생성 전 페이지 잔여 공간과 비교하여 사전 배치 결정
```

## STEP 7. 빈줄 규칙 (STRICT)

### 허용
```
① 표 뒤 자동 spacer: buildParagraphs가 다음 콘텐츠 레벨 기준으로만 삽입
② 그 외 텍스트 없는 빈 문단 추가 불허 — 분량·자간·내용 조정으로만 해결
```

### 표/풋노트 여백 원칙 (STRICT)
```
풋노트는 보충설명이 필요한 줄 바로 아래에 위치한다. 설명 대상과 풋노트 사이에는 빈줄을 넣지 않는다.

표도 본문 단락과 동일하게 처리한다:
  - 표 바로 아래 풋노트가 필요하면 TABLE → _isFn 순서로 붙인다.
  - 표 뒤 풋노트가 없으면 buildParagraphs가 다음 콘텐츠 레벨(H1/H2/body/sub)에 맞춰 자동 spacer를 삽입한다.
  - defs에 tinyGap() 또는 표 전용 빈줄을 직접 넣지 않는다.

금지 패턴:
  표 → tinyGap
  표 → tinyGap → 풋노트
  풋노트 앞 빈줄 삽입
```

### 금지
```
✗ 표 앞 빈줄 — 절대 금지 (이전 단락에 바로 붙임)
✗ H2·lv3·lv4 사이 임의 빈줄 — 금지
✗ 텍스트 없는 빈 문단 — 표 뒤 자동 spacer 외 금지
✗ 페이지 최상단 빈줄
```

### 중복 방지
```
H1 앞뒤로 h1Sep() 수동 삽입 금지 — buildParagraphs가 after 값으로 처리
표 뒤 tinyGap() 수동 삽입 금지 — buildParagraphs가 다음 레벨 기준으로 처리
```

## STEP 8. 계층 고립 방지 (분량 조정 우선)

**핵심 원칙: 분량 조정(내용 압축·보강)으로 계층 고립을 방지한다.**
keepNext 등 단락 옵션은 부서 문서에서 잘 사용하지 않는 방식이므로 사용하지 않는다.
페이지 넘어가는 현상은 최선을 다해 방어하되, 분량 조정으로 해결이 어려운 경우 허용한다.

### 계층 고립 방지 원칙

```
단락 옵션(keepNext, keepWithNext, pageBreakBefore) 사용 금지
→ 분량 조정(내용 압축·보강·여백 단락)으로만 해결

계층 고립 판단 기준:
  lv1·lv2가 페이지 끝 1~2줄에 단독으로 위치 → 고립
  해결: 이전 페이지 내용 1~2줄 보강하여 lv1·lv2를 다음 페이지 첫 줄로 이동
        또는 현재 섹션 내용 1~2줄 압축하여 lv1·lv2 + 하위 항목이 같은 페이지에 들어오도록

페이지 넘어가는 현상 방어 수위:
  최선을 다해 방어하되, 분량 조정 범위를 넘어서는 경우는 넘어가는 것 허용
  → 내용 품질을 해치면서까지 억지로 맞추지 않는다
```

## STEP 12. 페이지 정돈 — 3줄 기준 예외 처리

**페이지당 예상 단락 수 기준 (A4, 현재 여백 설정)**

```
본문(-) 단락 1개: 약 0.9~1줄
H1:               약 1.5줄 상당
H2:               약 1.2줄 상당
표 1행:           약 1.5~2줄 상당
페이지 전체:      약 22~25줄 (본문 단락 기준)
3줄 임계값:       약 3개 단락 또는 표 2행 이하
```

**케이스 A — 그룹/표의 3줄 이하만 다음 페이지로 넘어가는 경우**

```
문제: 그룹/표 대부분이 페이지 N에 있고, 마지막 3줄 이하만 N+1로 넘어감
해결: 그룹/표 직전 내용을 1~2줄 줄여서 그룹/표 전체를 N+1로 이동
     → 페이지 N이 조금 비어도 그룹/표 전체가 N+1에서 깔끔하게 시작

예)
  페이지 N:  [이전 내용 20줄] [표 헤더 + 3행] ← 표가 두 페이지에 걸림
  페이지 N+1: [표 마지막 2행]  ← 3줄 이하
  → 이전 내용을 1~2줄 압축 → 표 전체가 N+1로 이동
```

**케이스 B — 그룹/표 직전에 3줄 이하만 페이지 N에 남는 경우**

```
문제: 페이지 N 끝에 이전 내용 3줄만 남고, 그룹/표는 N+1에서 시작
해결: 이전 내용을 1~2줄 늘려서 페이지 N을 충분히 채우고,
     그룹/표는 N+1에서 자연스럽게 시작
     → 페이지 N 끝이 어색하게 비는 현상 방지

예)
  페이지 N:  [내용 19줄] [빈 공간 3줄]
  페이지 N+1: [표 시작]
  → 이전 내용을 1~2줄 보강 → 페이지 N이 꽉 찬 상태로 넘어감
```

**코드 전 페이지 계획 체크 (FLOW 2 페이지 계획 단계에서 수행)**

```
각 페이지별 예상 단락 수를 먼저 나열하고:
  1. 그룹/표가 페이지 경계 근처에 위치하는지 확인
  2. 3줄 이하 잔량 케이스 탐지
  3. 탐지된 경우 직전 내용 조정 계획 수립 후 코드 작성
```
